# Scad_for_stl

这是一个用 OpenSCAD 做 3D 打印模型的个人模型库。仓库里主要放两类东西：

- 可以直接打开、预览、导出 STL 的 `.scad` 模型文件。
- 做模型时反复用到的通用结构和小模块，比如盒子、螺纹、铰链、卡扣、蜂窝板、螺丝螺母等。

## 目录说明

```text
.
├── lib/              # 可复用的基础模块：盒子、盖子、螺丝柱、端子、开孔、电池盒等
├── z_通用结构/       # 通用结构示例/零件库：螺纹、蜂窝板、燕尾槽、铰链、挂钩等
├── z_万物模型/       # 具体实物模型：盒子、支架、门牌、磁铁字、电机固定架等
├── z_声学超结构/     # 声学/风道/共振器/降噪相关模型和实验结构
├── gears/            # 齿轮和传动相关模型
├── scripts/          # 辅助脚本
├── stls/             # 导出的 STL 文件，通常不提交
├── output_imgs/      # 导出的图片，通常不提交
├── svg/              # SVG 输入素材
└── document/         # 文档、记录或参考资料
```

## 依赖

大部分模型依赖 [BOSL2](https://github.com/BelfrySCAD/BOSL2)，部分旧模型或实验文件也可能依赖本仓库里的 `lib/` 模块。

使用前需要让 OpenSCAD 能找到 BOSL2。常见做法是把 `BOSL2` 放到 OpenSCAD 的库目录，或者放在当前项目能被 OpenSCAD 搜索到的位置。文件中常见写法如下：

```scad
include <BOSL2/std.scad>
include <BOSL2/threading.scad>
```

## 怎么使用

1. 用 OpenSCAD 打开需要的 `.scad` 文件。
2. 修改文件顶部或靠前位置的参数，比如尺寸、壁厚、孔径、螺距、数量等。
3. 按 `F5` 预览。
4. 按 `F6` 渲染。
5. 导出 STL 后再切片打印。

如果是要找可复用结构，优先看 `z_通用结构/`。如果是要做具体成品，优先看 `z_万物模型/` 和 `z_声学超结构/`。

## 常用入口

- `z_通用结构/003_螺纹.scad`：外螺纹和内螺纹螺母示例。
- `z_通用结构/010_螺丝螺母等.scad`：BOSL2 的螺丝、螺母样式展示。
- `z_通用结构/002_蜂窝板.scad`：蜂窝结构板。
- `z_通用结构/005_燕尾槽.scad`：燕尾槽连接结构。
- `z_通用结构/006_铰链.scad`：铰链结构。
- `lib/simple_box.scad`：基础盒体。
- `lib/lid.scad`：盖子相关模块。
- `lib/bolt_post.scad`：螺丝柱。
- `lib/port.scad`：接口/开孔相关模块。

## 文件命名约定

目前文件通常使用：

```text
编号_中文说明.scad
```

例如：

```text
003_螺纹.scad
017_带螺纹的盒子.scad
```

编号用于保持文件顺序，中文说明用于快速知道模型用途。`old/` 目录一般是旧实验或历史版本，重新做模型时可以参考，但不一定是当前推荐写法。

## 开发习惯

- 新的通用结构放到 `z_通用结构/`，尽量做成参数化模块。
- 新的具体模型放到 `z_万物模型/` 或对应主题目录。
- 能复用的基础结构放到 `lib/`，具体模型里用 `use <lib/xxx.scad>` 或 `include <lib/xxx.scad>` 引入。
- 导出的 `.stl`、图片、音频、表格等中间产物一般不提交，已经在 `.gitignore` 里忽略。
- 临时测试文件尽量用 `del.scad`、`del.py` 之类的名字，完成后再整理进正式目录。

## OpenSCAD 注意点

- 最后一个参数后面尽量不要多写逗号，某些情况下会导致解析问题。
- 默认参数里少用复杂的空数组/嵌套数组；如果要用，先单独测试。
- 布尔差集 `difference()` 里用于切除的物体可以略微加大 `0.01`，避免共面导致预览闪烁或导出异常。
- 3D 打印配合件要预留间隙。螺纹、卡扣、滑槽这类结构建议先小样测试，再打印完整件。
